-------
###### 1.Dockerfile
	1.就是用来docker镜像构建的命令脚本，执行顺序从上到下，每个指令都会创建提交一个新的镜像层，并提交
	2.每个保留关键字全大写
###### 2.执行命令
docker build -f dockerfile -t 镜像名 .
###### 3.参数

| FROM       | 指定基础镜像                            |
| ---------- | --------------------------------- |
| MAINTAINER | 维护者信息                             |
| RUN        | 构建时候需要运行的命令                       |
| ADD        | 添加内容                              |
| WORKDIR    | 设置工作目录                            |
| VOLUME     | 挂载的目录                             |
| EXPOSE     | 暴露端口配置                            |
| CMD        | 指定容器启动要运行的命令（只有最后一个会生效，可被替代）      |
| ENTRYPOINT | 启动要运行的命令（可以追加命令）                  |
| ONBUILD    | （触发指令）当构建一个被继承Dockerfile时候就会运行的指令 |
| COPY       | 类似ADD，文件拷贝到镜像中                    |
| ENV        | 设置环境变量                            |
|            |                                   |
|            |                                   |
|            |                                   |
